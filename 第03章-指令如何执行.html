本章目标：第二章你知道了 CPU 靠"取指-译码-执行"循环干活，也见到了 LDR/ADD 这些指令。这一章我们钻进一条指令内部——它的二进制到底怎么编码？CPU 怎么"看懂"它？程序为什么能有 if、for、函数调用这些"拐弯"？学完你要能读懂一小段 ARM 汇编，并理解每条指令在硬件里的确切动作。

3.1 为什么要单独讲"指令怎么执行"

你可能觉得第二章讲过取指译码执行了，够了。但有几个绕不开的问题，第二章只是点到：

  • 一条指令是一串二进制，CPU 凭什么从这串 0/1 里知道"这是加法、用 R0 和 R1"？——编码规则。
  • 程序为什么能 if/else 走不同的路、能 for 循环、能调函数再回来？——分支与 PC 的操纵。
  • 你写 read() 时那句"陷入内核"，在指令层面到底是哪条指令？——特权/系统指令。

这些是理解调度、中断、系统调用的硬件地基。不啃下来，后面讲"中断返回改 PC"、"上下文里保存 PC"你都只能背。

3.2 一条指令长什么样：机器码的编码

回到厨师类比：菜谱上的一行字，比如"把 2 号碗和 3 号碗的东西相加，放进 5 号碗"。这行字对厨师(CPU)来说不是中文，而是一串固定格式的二进制。指令的本质就是一串有固定"字段布局"的二进制。

以 ARM 的 32 位定长指令为例(RISC 定长的好处上一章说过)，一条 ADD 指令的 32 个 bit 被切成若干"字段"，每个字段管一件事：

ARM 经典 32 位 ADD 指令编码(简化示意):

 31    28 27  25 24 21 20 19  16 15  12 11        0
┌────────┬──────┬──────┬──┬──────┬──────┬───────────┐
│ 条件码  │ 类型  │操作码 │S │ Rn   │ Rd   │  操作数2   │
│ cond   │ 000  │ 0100 │0 │ 源1  │ 目标 │  源2/立即数 │
└────────┴──────┴──────┴──┴──────┴──────┴───────────┘
   4位     3位    4位   1  4位     4位      12位

含义: "满足cond条件时, 把 Rn 和 操作数2 相加, 结果写入 Rd"

你不需要背这个布局，但要抓住三个洞察：

  1. 指令 = 若干定宽字段的拼接。CPU 的译码器就是一堆逻辑门，按位切开这些字段，各送到对应部件。这就是"译码"的物理含义——把 32 个 bit 分派到该去的地方。
  2. 寄存器编号只需 4 位(2⁴=16，正好 R0~R15)。还记得第二章说"寄存器少是因为指令里编号位数有限"吗？这里就是证据——字段就这么宽，多了塞不下。
  3. 指令里能塞进去的"立即数"位数有限(这里 12 位)。这引出 3.9 节一个经典问题:大常数怎么办。

3.3 操作码与操作数

把上面的字段归纳，任何指令都由两部分组成，跟自然语言的"动词+宾语"一样：

部分回答什么菜谱类比例子
操作码 opcode干什么动作动词:"相加""搬运"ADD / LDR / B
操作数 operand对谁干、结果放哪宾语:哪几个碗R0, R1, R2

操作数可以是三种来源，这个分类非常重要(对应下一节的寻址方式)：

  • 寄存器：操作数就在某个小碗里，如 ADD R2, R0, R1。最快。
  • 立即数：操作数是个写死在指令里的常数，如 ADD R0, R0, #1(加 1)。# 表示立即数。
  • 内存地址：操作数在内存里，得先算出地址去取，如 LDR R0, [R1]。最慢。

3.4 寻址方式：怎么"找到"要操作的数

"寻址方式(Addressing Mode)"就是回答一个问题：指令里怎么描述"我要的那个数在哪儿"？厨师要拿食材，可以说"就用你手里那个"(寄存器)、"用编号写死的 5 号货架那袋"(直接)、"用你记的那个货架号去拿"(间接)。CPU 也有这几种"指路方式"：

寻址方式含义ARM 例子生活类比
立即数寻址数就写在指令里MOV R0, #5"直接用 5 这个数"
寄存器寻址数在某个寄存器MOV R0, R1"用你手上那碗"
寄存器间接寄存器里存的是地址，去那个地址取LDR R0, [R1]"按你记的货架号去拿货"
基址+偏移地址 = 寄存器 + 一个偏移量LDR R0, [R1, #4]"你记的货架号往后数第 4 格"
PC 相对地址 = 当前 PC + 偏移LDR R0, [PC, #8]"从我现在站的位置往前 8 步"

"基址+偏移"是重中之重，因为它是访问数组、结构体成员、栈变量的底层机制。arr[i] 就是"数组首地址(基址) + i×元素大小(偏移)"；结构体 p->field 就是"结构体地址 + 字段偏移"。你写的所有 .、->、[]，编译后都是这条寻址方式。第五、六章讲栈和函数调用时会反复用到。

3.5 取指-译码-执行的硬件细节

第二章给了三步的框架，这里补上"每一步硬件到底做了什么"，因为后面讲中断和流水线要用。

① 取指 Fetch
   PC 的值 → 送上"地址总线" → 内存把该地址的 32 位指令
   → 经"数据总线"送回 → 存入"指令寄存器 IR"
   同时: PC += 4 (ARM 一条指令 4 字节, 自动指向下一条)

② 译码 Decode
   译码器(逻辑门)切开 IR 的各字段:
   - opcode → 决定 ALU 做什么 / 是不是访存 / 是不是跳转
   - 寄存器编号 → 从寄存器堆读出对应的值
   - 立即数 → 提取并按规则扩展

③ 执行 Execute (+ 访存 + 写回)
   - 运算类: 值送 ALU → 算 → 结果写回目标寄存器
   - 访存类: ALU 先算出地址 → 读/写内存 → (读的话)写回寄存器
   - 跳转类: 把新地址写进 PC (下一次取指就从新处开始!)

注意最后一句："跳转"的物理本质，就是往 PC 里写一个新值。PC 一变，下一轮取指就从新地方开始——程序就"拐弯"了。整个控制流(if/for/函数/中断)全靠这一个动作。记住它，这是本章的灵魂。

3.6 状态标志与条件执行

程序要能判断"如果 a > b 就……"。CPU 怎么"判断"？它不会思考，它靠标志位(Flags)。

标志位从哪来

ALU 每做一次运算(尤其是比较/减法)，除了算出结果，还会顺手记下几个"副产品"到一个特殊寄存器(ARM 里在 CPSR/xPSR 状态寄存器里)：

标志名字什么时候置 1
ZZero 零标志运算结果为 0(常用于判断"相等")
NNegative 负标志结果最高位为 1(负数)
CCarry 进位无符号运算发生进位/借位
VoVerflow 溢出有符号运算溢出

比较的真相：其实是做减法

CMP R0, R1(比较 R0 和 R1)在硬件里做的是 R0 - R1，但只更新标志位、不保存差值。然后：

  • 若 R0 == R1，差为 0 → Z=1。
  • 若 R0 < R1，会借位 → 通过 N、C 的组合能判断出来。

所以 C 语言里 if (a == b) 编译成:CMP 做减法置标志 → 后面的跳转指令看 Z 标志决定跳不跳。CPU 的"判断"从来不是思考，而是"做个减法看标志位"。

3.7 分支跳转：程序为什么能拐弯

有了标志位，就能实现"有条件地改 PC"——这就是分支(Branch)。

C 代码:
  if (a > b)
      x = 1;
  else
      x = 2;

编译成指令(示意):
      CMP   R0, R1        ; 比较 a(R0) 和 b(R1), 置标志位
      BLE   .else         ; Branch if Less or Equal: 若 a<=b 就跳到 else
      MOV   R2, #1        ; (a>b 这条路) x=1
      B     .end          ; 无条件跳过 else
  .else:
      MOV   R2, #2        ; x=2
  .end:
      ...

看懂这段,你就理解了所有控制流:

  • 有条件跳转(BLE/BEQ/BNE...):看标志位,满足才把目标地址写进 PC。这是 if/while/for 的核心。
  • 无条件跳转(B):直接改 PC。用于 else 跳过、循环回头、goto。
  • 带返回的跳转(BL, Branch with Link):跳过去之前,先把"返回地址"存进 LR 寄存器——这是函数调用的关键,第六章详讲。

循环怎么实现?就是"跳转回头"。for(i=0;i<10;i++) 编译后:循环体结尾有一条 CMP i,#10 + BLT 循环开头,只要没到 10 就把 PC 改回循环体开头。循环 = 反复把 PC 往回改。

3.8 PC 相对寻址与位置无关代码(PIC)

一个进阶但很实用的点。跳转目标地址怎么写在指令里?两种思路:

  • 绝对地址:"跳到内存 0x8004 处"。问题:如果这段程序被加载到内存的另一个位置,地址就全错了。
  • 相对地址(PC 相对):"从我现在这条指令往后跳 20 字节"。好处:不管程序被放到内存哪里,相对偏移永远正确。

ARM 的分支指令用的正是 PC 相对。由此产生一个重要概念:位置无关代码(Position Independent Code, PIC)——代码不写死绝对地址,可以被加载到任意位置运行。

为什么你要关心?① Linux 的共享库(.so)必须是 PIC,因为同一个库会被不同进程加载到各自不同的地址;② 内核、bootloader 的重定位也依赖它;③ 面试问"为什么 .so 要编译成 -fPIC"就是考这个。相对寻址是 PIC 的硬件基础。

3.9 立即数：为什么大常数放不进指令

3.2 节说过,ARM 32 位指令里留给立即数的字段只有 12 位左右。问题来了:我要 x = 0x12345678 这么大的常数,12 位根本装不下,怎么办?

这是 RISC 定长指令的一个"甜蜜的烦恼"。CISC 可以用不定长指令直接跟一个大常数,RISC 不行。ARM 的解法有几种:

  1. 拆成几条指令拼:先 MOV 低 16 位,再 MOVT 填高 16 位,两条指令拼出 32 位大数。
  2. 放进"文字池"(literal pool),用 PC 相对 LDR 取:把大常数存在代码附近的一小块数据区,用 LDR R0, [PC, #offset] 加载。你反汇编时常看到函数末尾一堆 .word 0x12345678,就是文字池。

这解释了一个新手困惑:为什么反汇编里一个"赋值大常数"会变成两三条指令?因为立即数字段装不下,只能拼或去内存捞。这也是 RISC 相比 CISC 在"代码密度"上的一个代价。

3.10 从 C 到指令：一个完整例子

把本章串起来。看一个带循环、判断、数组访问的函数,理解它每一句对应什么:

C 代码: 求数组前 n 个元素的和
  int sum(int *arr, int n) {
      int s = 0;
      for (int i = 0; i < n; i++)
          s += arr[i];
      return s;
  }

编译成 ARM 指令(示意, 参数: arr在R0, n在R1):
      MOV   R2, #0          ; s = 0        (R2 当 s)
      MOV   R3, #0          ; i = 0        (R3 当 i)
  .loop:
      CMP   R3, R1          ; 比较 i 和 n              ← 3.6 标志位
      BGE   .done           ; if i>=n 跳出循环          ← 3.7 条件分支
      LDR   R12, [R0, R3, LSL #2] ; R12 = arr[i]
                            ; 地址=arr + i*4(int占4字节) ← 3.4 基址+偏移
      ADD   R2, R2, R12     ; s += arr[i]               ← ALU 运算
      ADD   R3, R3, #1      ; i++                        ← 3.3 立即数操作
      B     .loop           ; 回到循环开头                ← 3.7 无条件跳转(改PC)
  .done:
      MOV   R0, R2          ; 返回值放 R0(约定)
      BX    LR              ; 返回调用者(跳到 LR 存的地址) ← 第六章预告

逐行都能对上本章知识点。这就是"高级语言最终变成搬、算、跳"的实证。你以后调试卡在没有源码、只有汇编的地方时(内核 panic、RTOS 崩溃),就靠读这种代码定位问题。

3.11 特权指令与陷入内核

第一章说过用户态不能直接碰硬件,要"喊话"给内核。这个"喊话"在指令层面是什么?

普通指令 vs 特权指令

ARM 把指令分两类:普通指令(算术、搬运、跳转)人人能用;特权指令(改中断开关、改 MMU 配置、访问某些系统寄存器)只有内核态(EL1)能用。用户态(EL0)执行特权指令会直接触发异常——硬件层面的"越权保护"。

系统调用指令:SVC

用户程序想请求内核服务(读文件、分配内存),用一条专门的指令主动"敲门":ARM 上是 SVC(Supervisor Call,老名字 SWI)。执行 SVC 会:

用户态执行 SVC 指令
     │
     ▼  CPU 硬件自动:
  ① 切换到内核态(EL0 → EL1)
  ② 保存返回地址
  ③ 跳到内核预先设好的"异常入口地址"
     │
     ▼
  内核的系统调用处理程序接管, 看你要什么服务, 代你办
     │
     ▼  办完执行返回指令
  切回用户态(EL1 → EL0), 回到 SVC 的下一条继续

所以你在 Linux 里调 read(),C 库最终会执行一条 SVC(x86 上是 syscall 指令),这就是"陷入内核"的确切时刻。用户态和内核态的切换,是由这一条硬件指令 + CPU 的自动动作完成的,不是软件魔法。这条线索会在第一卷第七章(中断)、第二卷(异常/EL)、第五卷(系统调用)反复出现。

3.12 回扣：指令流与上下文

现在可以把"上下文"理解得更深了。一个任务在运行,本质是 CPU 沿着这个任务的指令流,一条条取指-执行,PC 不断前进/跳转。任务此刻的"现场"包括:

  • PC:指令流走到哪一条了(下一条取哪);
  • 通用寄存器:算到一半的中间值;
  • 状态寄存器(标志位):最近一次比较的结果——如果切换时不保存它,恢复后那个 if 就判断错了!
  • SP:栈用到哪了(第五章)。

很多人保存上下文时只记得存通用寄存器和 PC,忘了状态寄存器(标志位)。而恰恰是它记录着"上次比较的结果",不存的话,恢复任务后紧接着的条件跳转会走错路。这是上下文切换代码里一个真实的坑,也是面试进阶考点。你学完本章 3.6 才能真正理解为什么它必须被保存。

3.13 面试常问问题

Q1:一条指令由哪些部分组成?CPU 怎么"看懂"它?
操作码(干什么)+ 操作数(对谁干)。译码器按固定字段布局切开二进制,分派到 ALU、寄存器堆等部件。

Q2:程序的分支/循环在硬件上如何实现?
比较指令(CMP=做减法)更新标志位,条件跳转指令看标志位决定是否把目标地址写入 PC。循环就是把 PC 往回改。控制流的本质是"改 PC"。

Q3:CPU 怎么做"判断"(a>b)?
不会思考,靠 ALU 做减法产生的 Z/N/C/V 标志位,条件分支据此走不同路。

Q4:什么是位置无关代码?为什么 .so 要 -fPIC?
用 PC 相对寻址、不写死绝对地址的代码,可被加载到任意地址运行。共享库会被不同进程映射到不同地址,必须 PIC 才能共享同一份物理代码。

Q5:用户态怎么进内核态?哪条指令?
主动:执行 SVC(ARM)/syscall(x86)系统调用指令,CPU 自动切态并跳到内核异常入口。被动:中断/异常。

Q6:上下文切换为什么要保存状态寄存器(标志位)?
它记录最近比较结果,不保存则恢复后的条件跳转会判断错误。

3.14 常见误区

  1. "CPU 能理解 if 的逻辑"——不能。它只会"减法置标志 + 看标志跳转"。逻辑是编译器翻译出来的。
  2. "跳转是个神秘操作"——不神秘,就是往 PC 写个新值而已。
  3. "立即数想多大写多大"——RISC 指令里立即数字段有限,大常数要拆指令或用文字池。
  4. "CMP 会算出差值存起来"——不存,只更新标志位。
  5. "用户态和内核态是软件概念"——是 CPU 硬件的状态,由特定指令和异常机制切换。
  6. "上下文只有通用寄存器和 PC"——还必须包括状态寄存器(标志位)和 SP。

3.15 本章总结

  • 指令 = 定宽字段拼成的二进制;译码就是逻辑门按字段切开分派。寄存器编号、立即数的位数都受字段宽度限制。
  • 指令 = 操作码(动词) + 操作数(宾语);操作数来自寄存器/立即数/内存。
  • 寻址方式回答"数在哪";其中"基址+偏移"是数组、结构体、栈变量的底层机制。
  • 执行阶段里,跳转的本质就是往 PC 写新值——所有控制流的根。
  • CPU 靠 ALU 产生的 Z/N/C/V 标志位做"判断";CMP 其实是做减法只置标志;条件分支看标志改 PC。
  • PC 相对寻址→ 位置无关代码(PIC)→ 共享库/-fPIC。
  • 大立即数放不进指令,要拆指令或用文字池。
  • 特权指令只有内核态能用;SVC/syscall 是用户态陷入内核的确切指令。
  • 回扣:任务现场 = PC + 通用寄存器 + 状态寄存器(标志位!) + SP,切换时一个都不能漏。

│  ├─ 寄存器编号位数有限→寄存器少
│  └─ 立即数字段有限→大常数放不下
├─ 二、操作码(动词)+操作数(宾语)
│  └─ 操作数三源:寄存器/立即数/内存
├─ 三、寻址方式(数在哪)
│  ├─ 立即/寄存器/寄存器间接
│  └─ ★基址+偏移=数组/结构体/栈变量
├─ 四、取指译码执行硬件细节
│  └─ ★跳转=往PC写新值(控制流之根)
├─ 五、标志位 Z/N/C/V
│  ├─ ALU运算的副产品
│  ├─ CMP=做减法只置标志
│  └─ CPU"判断"=减法+看标志
├─ 六、分支跳转
│  ├─ 有条件(if/while)/无条件(else/循环)
│  ├─ BL 存返回地址到LR(函数调用预告)
│  └─ 循环=把PC往回改
├─ 七、PC相对寻址→位置无关代码(PIC)
│  └─ .so 需 -fPIC
├─ 八、大立即数:拆指令/文字池
├─ 九、特权指令 & 陷入内核
│  └─ SVC/syscall:用户态进内核的指令
└─ 十、回扣:上下文=PC+通用寄存器+标志位+SP
