一、计算机组成与内存 第一卷

Q1. 讲讲存储金字塔,为什么需要 Cache?

寄存器→Cache(L1/L2/L3)→内存→磁盘,越往下越大越慢越便宜。CPU 和内存速度差几十倍("内存墙"),Cache 是中间的高速缓冲:利用局部性原理(时间/空间局部性),把常用数据放在快的 Cache 里,让大部分访问命中 Cache,掩盖内存的慢。核心思想是"用快的少量缓存慢的大量"。

Q2. Cache 未命中有几种?写策略?

三种 miss:强制(首次)、容量(装不下)、冲突(映射冲突)。写策略:写回(write-back,先写 Cache 标脏,后回写内存,快)vs 写直达(write-through,同时写内存,简单但慢);写分配 vs 写不分配。嵌入式常用写回。

Q3. 虚拟内存解决什么问题?MMU 干什么?

虚拟内存给每个进程独立的地址空间,解决:①隔离(进程互不干扰、不能访问别人内存)②灵活(物理内存不连续也能给进程连续地址)③按需分配(用到才给物理页)。MMU 负责虚拟地址→物理地址的翻译(查页表),TLB 缓存翻译结果加速。

Q4. 什么是缺页?malloc 后立即用会怎样?

访问的虚拟页没有对应物理页时触发缺页异常,内核分配物理页填页表。malloc 只给虚拟地址(不给物理页),首次访问才缺页从 Buddy 分物理页(按需分页)。所以 malloc 成功不代表有物理内存,写时才真正分配(还有 COW)。

Q5. COW(写时复制)是什么?

fork 后父子进程共享同一物理页(只读),谁写才复制一份该页给它。避免 fork 时立即拷贝整个地址空间(大多数 fork 后马上 exec 换掉,拷了白拷)。省内存、fork 快。

二、ARM 架构 第二卷

Q1. Cortex-M 和 Cortex-A 的区别?

M 系列:微控制器,无 MMU(只有 MPU)、面向实时/低功耗、跑 RTOS/裸机(飞控 MCU);A 系列:应用处理器,有 MMU、跑 Linux/Android、性能强(RK3588 的 A76/A55)。M 确定性好,A 功能强。

Q2. ARM 的异常等级 EL0-EL3 是什么?

特权分级:EL0 用户态(应用)、EL1 内核态(OS 内核)、EL2 虚拟化(Hypervisor)、EL3 安全监控(Secure Monitor/固件)。越高权限越大。系统调用是 EL0→EL1,TrustZone 切换经 EL3。

Q3. GIC 是什么?中断类型?

ARM 的通用中断控制器,管理中断的分发。组成:Distributor(全局分发)、Redistributor/CPU Interface(送给核)。中断类型:SGI(核间软中断)、PPI(核私有)、SPI(共享外设中断)、LPI(基于消息 MSI)。

Q4. 什么是内存屏障?为什么需要?

CPU/编译器会乱序执行/重排内存操作(为性能)。多核下这会导致其他核看到"不一致"的顺序。内存屏障(DMB/DSB/ISB)强制某些内存操作的顺序和可见性。锁的 acquire/release 语义就靠屏障保证临界区不越界。

Q5. MESI 缓存一致性协议?

多核每个核有自己 Cache,MESI 用四个状态(Modified/Exclusive/Shared/Invalid)维护一致性:一个核改了某 Cache 行,通过总线嗅探让其他核的该行失效,保证不会读到旧值。ARM 用 MOESI/类似协议(ACE/CHI)。

Q6. TrustZone 是什么?

ARM 的硬件安全隔离:把系统分成安全世界(Secure)和普通世界(Normal),用 NS 位标识,经 SMC 指令+EL3 监控切换。安全世界跑 TEE(如 OP-TEE),放密钥/指纹/DRM 等敏感操作,普通世界(Linux)访问不了。

三、FreeRTOS 源码 第三卷

Q1. FreeRTOS 任务创建到运行的过程?

xTaskCreate 分配 TCB 和栈,pxPortInitialiseStack 伪造初始栈帧(假装任务被中断过,栈里放好初始寄存器和任务入口),把任务加入就绪链表。调度器启动后,上下文切换时从栈恢复出这个"伪造现场",任务就开始执行了。

Q2. FreeRTOS 上下文切换怎么实现?PendSV?

用 PendSV(可挂起系统调用,最低优先级异常)做上下文切换:需要切换时触发 PendSV,等所有中断处理完才执行它,在 PendSV 汇编里保存当前任务寄存器到其栈、切换 TCB、从新任务栈恢复寄存器。放最低优先级是为了不打断真正的中断。SysTick 提供时基。

Q3. FreeRTOS 调度算法?怎么选下一个任务?

固定优先级抢占式:总是运行最高优先级的就绪任务,同优先级时间片轮转。用优先级位图+CLZ 指令(计算前导零)快速找最高优先级,O(1)。这就是 RMS 思想(周期短设高优先级)。

Q4. FreeRTOS 队列/信号量/互斥量的关系?

"一切 IPC 皆队列":信号量、互斥量底层都是队列的特例。队列传数据(带拷贝);信号量是不传数据只计数的队列(同步/资源计数);互斥量是带优先级继承的二值信号量(保护临界区)。要互斥保护用 Mutex(有优先级继承防反转),不用二值信号量。

Q5. FreeRTOS 的堆管理有几种?

heap_1(只分配不释放)、heap_2(可释放但不合并,有碎片)、heap_3(包装标准 malloc)、heap_4(可释放+合并相邻空闲块抗碎片,最常用)、heap_5(支持多块非连续内存)。实时系统重确定性,常用 heap_4 或内存池。

四、RT-Thread 第四卷

Q1. RT-Thread 的内核对象机制?

C 语言实现的"面向对象":rt_object 是基类(有类型、名字、链表节点),线程/信号量/设备等都"继承"它(把 rt_object 作为结构体第一个成员,C 式继承)。对象容器统一管理所有内核对象。体现了用 C 模拟 OOP。

Q2. RT-Thread 设备驱动框架?

rt_device 统一抽象所有设备(GPIO/UART/SPI...),用函数指针表(rt_device_ops:init/open/read/write/control)实现多态,各设备各自实现。应用用统一 API(rt_device_read/write)操作任何设备——"一切设备皆 rt_device",是 Linux VFS 的迷你版。

Q3. RT-Thread 和 FreeRTOS 的区别?

FreeRTOS:极简、只有内核(任务/IPC/内存),生态靠自己搭;RT-Thread:更丰富,自带设备框架、FinSH shell、文件系统、网络协议栈、软件包生态,更像"小型 Linux"。RT-Thread 是从 RTOS 通向 Linux 的桥梁。

五、Linux 内核 第五卷

Q1. 进程和线程的区别?Linux 怎么实现?

进程有独立地址空间,线程共享地址空间。Linux 里都用 task_struct 表示,用 clone 创建:fork(进程,不共享地址空间,COW)、pthread(线程,共享地址空间/文件表等)。Linux 线程是"共享资源的进程",调度单位都是 task_struct。

Q2. CFS 调度器原理?

完全公平调度:用 vruntime(虚拟运行时间)记录每个任务已运行的加权时间,总是选 vruntime 最小的运行(用红黑树 O(log n) 找),让所有任务公平推进。nice 值影响权重(高优先级 vruntime 增长慢、跑得多)。追求公平/吞吐,非实时。

Q3. Linux 内核内存分配的两层?

底层 Buddy(伙伴系统)管物理页:按 2^order 分组,分配对半分裂、释放合并伙伴,抗外部碎片。上层 Slab/SLUB 管小对象:为每种对象建缓存(=按大小分类的内存池),O(1)、无碎片、per-CPU 快。Buddy 批发页、Slab 零售对象。

Q4. kmalloc 和 vmalloc 的区别?

kmalloc:物理连续(能给 DMA 用),底层 Slab/Buddy,受大块碎片限制;vmalloc:只虚拟连续(物理零散页拼,页表映射),能分很大但不能给物理连续 DMA、有页表开销。DMA 用 kmalloc。

Q5. VFS 的作用?fd、file、inode 的关系?

VFS 是文件系统抽象层,统一 read/write 接口(一切皆文件)。fd(进程私有整数)→查 fd 表→file(打开会话,有读写位置 offset)→inode(文件实体)。dup 共享 file(共享 offset);两次 open 共享 inode(不共享 offset);fork 复制 fd 表但共享 file。

Q6. 信号是什么?何时递送?

内核给进程的异步通知,本质是"软件中断"(进程层的中断)。产生(设未决位)→挂起(pending,被 blocked 掩码可挡)→递送(在进程从内核态返回用户态时处理)。所以能打断阻塞的 read(返回 EINTR)。SIGKILL/SIGSTOP 不可捕获。

Q7. epoll 为什么比 select 高效?

select 每次全量传 fd 集合、内核 O(n) 遍历全部;epoll 用 epoll_ctl 登记一次存内核红黑树,数据到达靠回调把就绪 fd 放就绪链表,epoll_wait 只取就绪的 O(就绪数)。性能与就绪数挂钩而非总数,能撑百万连接。LT(默认)vs ET(边沿,须循环读到 EAGAIN)。

Q8. 中断的上半部/下半部?为什么?

ISR 要短(期间可能关中断,久了误事),但有些中断有重活。所以拆:上半部(硬中断)极短,应答硬件+触发下半部;下半部延后做重活。下半部三机制:SoftIRQ(不睡,多核并行,核心子系统)、Tasklet(不睡,串行化,普通驱动)、Workqueue(内核线程,能睡)。

Q9. 自旋锁和互斥锁的区别?

自旋锁:拿不到锁忙等(不睡,占 CPU),适合短临界区、可用于中断上下文,持锁不能睡;互斥锁:拿不到锁睡眠(让 CPU,有切换开销),适合长临界区、只能进程上下文。中断上下文/短临界区→自旋锁,长临界区→互斥锁。进程和中断共用的自旋锁,进程侧要 spin_lock_irqsave(关中断防死锁)。

Q10. RCU 是什么?怎么做到读者无锁?

Read-Copy-Update,读多写少场景让读者几乎零开销。写者拷贝副本改好、原子切指针(新旧共存),等所有老读者离开再释放旧数据。因读者临界区内不能睡眠,"每个 CPU 都上下文切换过一次"就说明老读者都退出了(宽限期)——把"检测读者"转成"观察 CPU 调度",不打扰读者。

六、Linux 驱动 第六卷

Q1. 写字符驱动的核心?调用链?

核心是实现 file_operations(read/write/open/ioctl)并注册。调用链:应用 read→VFS file->f_op->read 多态分发→你的驱动 read 函数→操作硬件。驱动和用户交换数据必须用 copy_to_user/copy_from_user(不同地址空间,用户指针不可信)。

Q2. 总线-设备-驱动模型?probe 何时调用?

用 Bus(撮合)、Device(设备信息)、Driver(驱动逻辑)分离,让一套驱动适配不同硬件。总线按匹配规则(compatible/VID-PID)撮合设备和驱动,匹配成功调 probe。probe 里从设备树取寄存器地址/中断/时钟,映射寄存器、初始化硬件、注册设备。

Q3. 设备树是什么?三大关键属性?

描述硬件的文本文件(.dts),把硬件描述从内核代码剥离成数据,让通用内核适配海量硬件。三大属性:compatible(匹配驱动)、reg(寄存器地址+大小)、interrupts(中断)。换板只换设备树,驱动不改。

Q4. DMA 的 Cache 一致性问题?(高频重点)

DMA 直接访问内存绕过 CPU Cache,导致不一致:DMA 写后 CPU 可能从 Cache 读到旧值;CPU 写后数据在 Cache 没回写,DMA 读到旧值。解决:DMA 接收前 invalidate Cache、发送前 flush Cache。Linux DMA API(dma_map_single/dma_alloc_coherent)自动处理。

Q5. UART/I2C/SPI 怎么选?

UART:异步(无时钟线,靠波特率)、点对点、简单,调试台/GPS/模块;I2C:同步、两线、多设备靠地址、低速,省引脚接多个传感器;SPI:同步、四线、全双工、高速、靠片选,高速 IMU/Flash/屏。按速度、设备数、引脚预算选。

Q6. I2C 读传感器寄存器的时序?

START→发[设备地址|写]+ACK→发寄存器号+ACK→RESTART→发[设备地址|读]+ACK→读数据→STOP。即"先写告诉它读哪个寄存器,再转向读值"。ACK 缺失是调试关键(设备没接好/地址错/没上拉)。I2C 要接上拉电阻(开漏)。

Q7. SPI 的 CPOL/CPHA?串口的规范/原始模式?

SPI:CPOL(时钟空闲电平)+CPHA(采样边沿)组成 4 种模式,主从必须一致否则读乱数据(头号坑)。串口 tty:规范模式(行编辑/回显/Ctrl+C→信号,人用命令行)vs 原始模式(透传);接传感器传数据必须 cfmakeraw 切原始模式,否则数据被终端逻辑篡改。

Q8. CAN 的非破坏性仲裁?(车企重点)

逐位仲裁:节点边发 ID 边监听,显性(0)盖过隐性(1);发现自己发 1 但线上是 0,说明有更高优先级(ID 更小)在发,立即退出,赢家报文不受影响。保证最高优先级消息确定性胜出(实时),vs 以太网 CSMA/CD 破坏性退避。CAN 面向消息、差分抗干扰、Linux 用 SocketCAN。

Q9. mmap 为什么零拷贝?

通过操作页表让用户虚拟地址和内核/设备物理页指向同一块物理内存,用户直接访问=访问内核缓冲,无需 copy_to_user 拷贝。摄像头场景:DMA 把图像写进物理缓冲,mmap 映射给用户直接处理,全程 CPU 不搬数据。V4L2=ioctl 配参数+mmap 传图像。

七、实时系统 第七卷

Q1. 什么是实时性?实时就是快吗?(必考)

不是。实时=确定性/准时——在 deadline 内一定完成,最坏情况有保证。看最坏延迟有没有上界,不是平均快。平均慢但每次按时,比平均快但偶尔超时更实时。"迟到的正确结果=错误结果"。硬实时(错过=灾难,飞控)vs 软实时(可容忍,视频)。

Q2. 延迟从哪来?怎么降?

响应延迟=中断延迟+ISR+调度延迟+任务执行。主要不确定源:关中断时间(头号隐形杀手,临界区要短)、不可抢占段、优先级反转、换页/Cache。降低:短临界区、PREEMPT_RT、优先级继承、mlockall 锁内存、绑核隔离。用 cyclictest 测最坏延迟。

Q3. RMS 和 EDF?可调度性?

RMS(固定优先级):周期短=优先级高,最优固定优先级算法,判据 U≤n(2^(1/n)−1)≈69%。EDF(动态优先级):deadline 近=优先级高,判据 U≤1(利用率 100%),但过载会雪崩。安全系统(飞控)多用 RMS(可预测、过载优雅降级),EDF 利用率高但复杂。

Q4. 优先级反转?怎么解决?(必考)

高优先级任务被低优先级任务(持共享锁)阻塞。无界反转:中优先级任务抢占持锁的低优先级,使高优先级等待不可控——摧毁实时性(火星探路者号事件)。解决:优先级继承(临时把持锁的低优先级提到高优先级,尽快释放)/优先级天花板(一拿锁就提到天花板,还防死锁)。FreeRTOS Mutex 带继承,信号量不带。

Q5. WCET 是什么?为什么难求?

最坏执行时间,可调度性分析的输入。难求因:路径爆炸、循环次数依赖输入、最主要是硬件不确定(Cache 命中/未命中差几十倍且依赖历史、流水线、乱序)。现代 CPU 优化(为平均性能)恰是 WCET 敌人。只能求安全上界。为可预测编程:循环有静态上界、无递归、无 malloc。

Q6. PREEMPT_RT 做了什么?

把 Linux 改造成几乎完全可抢占。核心:①自旋锁→可睡眠可抢占的 rt_mutex(消除不可抢占段+优先级继承)②中断线程化(中断处理放内核线程,可被实时任务抢占)。把最坏延迟从毫秒级降到 <100us 有界。但仍不及专用 RTOS,极致硬实时用 AMP。

八、多核/异构/工程 第八卷

Q1. SMP 和 AMP 的区别?(必考)

SMP(对称):多个对等核共享内存跑同一个 OS,统一调度,简单但隔离弱。AMP(非对称):不同核跑不同的独立 OS(如 Linux+RTOS),各资源独立、互不干扰,通过核间通信交互,隔离强、实时极好但复杂。AMP 解决"既要 Linux 功能又要 RTOS 硬实时",是车企主流架构。

Q2. 绑核和 CPU 隔离的区别?为什么绑核?

绑核(affinity):限制任务只在某核跑(但别人还能来干扰);隔离(isolcpus):把核从通用池拿走(别人不来)。两者+nohz+中断亲和=实时任务独占干净核。绑核好处:保持 Cache 亲和(避免迁移导致 Cache/TLB 重建)、执行时间确定。

Q3. 核间通信怎么实现?RPMsg?

物理基础=共享内存(传数据)+核间中断(通知)。RPMsg 提供消息抽象(像 socket),底层是 virtio/vring(共享内存里的环形队列=FreeRTOS Queue 跨核版)。RemoteProc 管理协处理器生命周期(加载固件/启停/恢复)。关键坑:共享内存 Cache 一致性(写 flush、读 invalidate)。

Q4. 零拷贝怎么做?为什么重要?

数据只存一份、各处理者共享同一物理内存、传句柄不传数据。技术:mmap(内核↔用户)、DMA-BUF(设备↔设备,传 fd,视觉流水线)、共享内存(进程间)。一帧 16MB 图像×30 帧拷贝几次=每秒 GB 级,零拷贝才能跑多路 4K。要处理 Cache 一致性和同步(多缓冲+队列)。

Q5. ROS2 能做硬实时吗?系统怎么分层?

ROS2 更面向实时(DDS 实时 QoS)但本身不保证硬实时。典型架构:Linux 大核跑 ROS2(感知/SLAM/规划,DDS+零拷贝+NPU+PREEMPT_RT)+RTOS 专核跑硬实时控制(姿态/电机 FOC/CAN),两侧 RPMsg+共享内存协作(AMP)。硬实时飞控放 RTOS,不放 ROS2。

Q6. ROS2 为什么用 DDS?QoS?

DDS 是工业级数据分发中间件:去中心化自动发现(无 master 单点故障,比 ROS1 鲁棒)、QoS、实时。QoS 核心 Reliability:关键指令 Reliable(可靠重传)、高频传感器流 Best Effort(尽力低延迟)——就是 TCP/UDP 取舍。发布订阅 QoS 要兼容否则连不上。

九、答题策略与项目复盘

答题结构(STAR-T)

回答技术题的黄金结构:①先给结论/定义(一句话)→②讲为什么/解决什么问题→③讲底层怎么实现→④举例/联系你的项目→⑤主动延伸一层(体现深度)。面试官爱追问"为什么",你主动往下讲一层,能大大加分。

项目复盘要点

面试一定会深挖你的项目。用本书知识重新武装你的项目描述:

  • STM32/RTOS 项目:能讲清任务怎么分优先级(RMS 思想)、用了什么 IPC、临界区怎么保护、有没有遇到优先级反转、内存怎么管(内存池 vs malloc)。
  • Linux 应用/驱动项目:能讲清用了什么驱动模型、怎么和内核交换数据(copy_to_user/mmap)、中断怎么处理(上下半部)、并发怎么同步(锁/RCU)。
  • RK3588/ROS2 项目:能讲清任务怎么分配到核(绑核/隔离)、大数据怎么传(零拷贝/DMA-BUF)、实时任务怎么保证(PREEMPT_RT/AMP)、ROS2 通信怎么配(话题/QoS)。
  • 调试经历:准备 1-2 个"排查诡异 bug"的故事——如 DMA Cache 一致性导致数据错乱、优先级反转导致偶发卡顿、串口没切原始模式丢数据、QoS 不兼容收不到消息。这些"踩坑+定位+解决"的故事最能体现工程能力。

核心:把"我做过 X"升级成"我做过 X,底层原理是 Y,遇到过 Z 问题,这样解决的"。本书就是帮你把项目经历从"会用"提升到"懂原理"。

贯穿全书的"大思想"(面试展现体系感)

大思想在哪些地方体现

十、必背速查卡

面试前 10 分钟过一遍这些"一句话要点":

【内存】
 Cache = 用快的少量缓存慢的大量(局部性)
 虚拟内存 = 隔离+灵活+按需; MMU翻译, TLB加速
 malloc不给物理页, 访问才缺页分配; COW = 写才复制
 Buddy(页,合并伙伴抗外碎片) + Slab(对象池,无碎片)
 kmalloc物理连续(DMA) / vmalloc虚拟连续

【ARM】
 EL0用户/EL1内核/EL2虚拟化/EL3安全
 GIC: SGI核间/PPI私有/SPI外设/LPI(MSI)
 内存屏障DMB/DSB/ISB; MESI一致性; TrustZone安全世界

【FreeRTOS】
 伪造初始栈帧 → PendSV(最低优先级)做上下文切换
 固定优先级抢占(RMS); 一切IPC皆队列
 Mutex带优先级继承(信号量不带); heap_4合并抗碎片

【Linux内核】
 进程/线程都是task_struct, clone创建
 CFS: vruntime最小者跑, 红黑树, 追求公平
 信号=软件中断, 返回用户态时递送
 epoll = 登记一次+回调推送就绪(O(就绪数))
 上半部短/下半部(softirq不睡/workqueue能睡)
 自旋锁(忙等/不睡/中断上下文) vs 互斥锁(睡/进程上下文)
 RCU: 读者无锁, 宽限期=每CPU切换过一次

【驱动】
 填file_operations + 注册; copy_to/from_user
 总线-设备-驱动: compatible匹配→probe
 ★DMA一致性: 收前invalidate, 发前flush
 I2C要上拉/ACK; SPI主从CPOL/CPHA一致; 串口传数据用原始模式
 CAN非破坏仲裁(ID小优先, 显性盖隐性)
 mmap零拷贝 = 页表指向同一物理内存

【实时】
 实时=确定性(最坏有保证), 非快
 关中断=头号隐形杀手; 临界区要短
 RMS周期短优先(≈69%) / EDF deadline近(100%但雪崩)
 优先级反转→继承/天花板(火星探路者)
 WCET难因硬件不确定; PREEMPT_RT: 自旋锁可睡+中断线程化

【多核/工程】
 SMP(一OS对等核) vs AMP(多OS各管核, Linux+RTOS)
 绑核(Cache亲和) + isolcpus隔离 = 独占干净核
 核间通信 = 共享内存 + 中断; RPMsg/vring = 跨核队列
 零拷贝 = 数据一份共享, 传句柄(DMA-BUF/共享内存)
 ROS2: 节点+话题(pub/sub)+DDS+QoS; 硬实时放RTOS(AMP)
